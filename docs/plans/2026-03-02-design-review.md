## 第二轮审核（v2 文档）

> v1 的 17 个问题和 8 个待答问题全部得到回应。以下为 v2 新发现的问题。

### 问题 1：sessionStorage 关闭标签页即丢失 token（建议修复）

`sessionStorage` 在关闭标签页时清除。场景：

- 玩家游戏中不小心关闭标签页
- 立即重新打开（10 秒内）
- token 已丢失，无法重连
- 60 秒重连窗口形同虚设

**建议：** 改用 `localStorage` 存 token（与昵称一致）。如果担心多标签页冲突，可以 `localStorage` 存 token + `sessionStorage` 存"活跃标签"标记来区分。

### 问题 2：游戏中主动 `room:leave` 的处理未定义（建议修复）

断线有 60 秒重连窗口，但**主动**点"离开房间"是不同的场景：

- 主动离开 = 主动放弃，不应该有等待窗口
- 应该立即判负，游戏结束

**建议：** 在设计文档中补充一句：游戏中 `room:leave` = 立即判负，游戏结束。

### 问题 3：叫分"一轮"定义不够精确（低优先级）

设计说"后续玩家可叫更高分数"和"一轮结束后，最高分者为地主"，但未明确：

- 每人是否只有一次叫分机会？（标准规则是一次）
- 还是像拍卖一样可多轮加价？

**建议：** 明确写出"每人按顺序有且仅有一次叫分机会"。

### 问题 4：`getFullState(playerId)` 返回结构未定义（低优先级）

Task 10 提到此方法但未定义返回类型。实现时需注意：

- 只包含该玩家自己的手牌，其他玩家只返回牌数
- 需包含：阶段、轮次、地主信息、上一手牌、底牌、叫分进展、得分状态

**建议：** 在 `types/events.ts` 中定义 `GameSnapshot` 类型。

### 问题 5：CORS 配置未提及（低优先级）

v1 有 `cors: { origin: '*' }`，v2 未提。MVP 阶段可以用 `*`，但应在 server 配置中明确写出。
